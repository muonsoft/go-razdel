# AGENTS Guidelines

## Scope

Этот репозиторий предназначен для поэтапного переноса `natasha/razdel` на Go с контролем поведенческой совместимости.

## Working Rules

1. Не изменять `third_party/razdel` без явной задачи на обновление upstream.
2. Любое обновление submodule делать отдельным коммитом с указанием старого и нового commit SHA.
3. При реализации ориентироваться на behavioral parity с upstream-реализацией.
4. Делать небольшие, проверяемые инкременты вместо крупных одномоментных изменений.
5. Спорные решения (границы токенизации, Unicode edge cases, формат API) фиксировать в `README.md` или отдельных notes-файлах.
6. На bootstrap-этапе не переносить алгоритмы из Python в Go.

## Validation Expectations

- Для каждого изменения добавлять или обновлять тесты.
- Проверять локально минимальный набор команд:
  - `go test ./...`
  - `go vet ./...`
  - `golangci-lint run ./...`

## Definition of Ready for Implementation Tasks

- Есть четкая ссылка на upstream-поведение (файл/пример/тест).
- Изменение ограничено конкретной, измеримой целью.
- Критерии приемки выражены через тесты или воспроизводимые примеры.
